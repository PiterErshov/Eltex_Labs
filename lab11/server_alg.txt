Альгоритм сервера.

1. Сервер создаёт один поток с броадкаст-сокетом.
2. Бродкаст-сокет шлёт ip сервера и порт для подключения всем в подсети.
3. Сервер создаёт ещё один поток с сокетом, который ждёт подключения от клиентов.
4. Сервер посчитывает число клиентов и создаёт по из числу потоки с сокетами для обмена информацией.
5. Сервер ждёт, когда клиенты отправят свои сообщения.
6. Если клиенты прислали сообщение с флагом на закрытие сокета, то сокет закрывается и поток заверашется.
7. Сервер работает пока не получит команду на отключение.
